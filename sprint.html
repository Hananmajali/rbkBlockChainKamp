MetaMask:

- At the begining you have to download metamask from Chrome Web Store.
1- in your Chrome browser navigate to Chrome web store
2- search for metamask.
3- click ADD TO CHROME.
4- you will have its icon in the top right of your browser.
5- click on the icon, and complete the steps untill you have the "create an account" window.
6- enter your password and save your secret words.

- After that you will have your account (Account1) created, you can see that you don't have any ethers until now.
- From the drop list on the top you can see the Ethereum networks :
a- The main network.
b- The test networks :
1- Ropsten.
2- Kovan.
3- Rinkeby.
c- Local host 8545 : when we are hosting a local Ethereum node or local Ethereum test network.
d- Custom RPC : when we are using some remote network at some address.

- Now you have to get some ethers :
1- for that we will use Rinkeby test network, select it from the drop list in the top.
2- copy your account address, click on the three dots beside your account name (Account1) and then select copy.
3- in your browser navigate to faucet.rinkeby.io
4- follow the steps mentioned in the web site to get your ethers.
5- after completing all the steps successfully, your balance should be changed and you should see the amount of ethers you get.
Notice that if you changed the network, your balance will not be the same.

- Perform a transaction :
1- create another account from the icon on the top right, click on it and select Create Account.
2- now you have two accounts (Account1 and Account2), you can switch between them by selecting the account name from the same icon.
3- from Account1, click on send and put the recipient address (Account2) and yhe ammount you want to transfer, then click next then submit.

Notice that the balances will not change immediately, you will be notified when it is completed, after that you can check the balance for the both accounts.

I. Warm Up exercise


pragma solidity ^0.4.0;
contract exercise {
   uint public balance;
   string  name= "Sara"
   int[] public nums=[1,2,3];
   function setBalance(int x) public {
       balance = x;
   }
   function getBalance() public  returns (uint) {
       return balance;
   }
   function doubleBalance() public view returns (uint) {
       balance=2*balance;
       return balance;
   }
   function tripleBalance() constant public returns (uint) {
       balance=3*balance;
       return balance;
   }
}

1- Fix the errors.
2-In the run tab, deploy your contract and click on your deployed contract, set you balance.
3-Try getBalance method, did you get your balance ?search about what can you fix to get your balance from this method.
4-Do you really need getBalance method or you can get it in another way? Why cant you get name the same way you get balance?
5-Try getting num array without writing a getter for it. Did it work?
6- Double your balance and check it again, is it doubled? Search for the reason.
7- Triple your balance, what is the returned value and why? Search for the reason.
8-Write a method that uses another method in the contract and make sure of your result.

II. BlockChain Camp Database:

Create a contract BCCamp that will store information of BlockChain mini boot camp students, this information includes name, age, and if he/she is enjoying or not :D and every student information should be bound with his/her ethereum account address. You need to build these functions:

1. addStudent which given information of a student and his/her address will add it to the database.
2. getStudent which returns an information of a student when it’s given the address of this students (Note: don’t use for loops in this exercise, since you need to pay for every iteration in the for loop, for more information see gas spending in solidity.Also if used the best data type in 1 you can't iterate over its elements.)
Bonus:
3. getStudents that should return all of the registered students addresses.
4. countStudents that should return how many students are registered.

III. Auction Contract:

Create a contract Auction that will receive bids from people who are willing to get a specific asset, the asset in our case will be only a password given from the beneficiary to the bidder who offers the highest price after the period of the auction time is over. Your job is to build the following functions:

1. Constructor that stores the address of the beneficiary and the password of the asset to be bought, and sets the auction duration to 3 hours from the time of deployment.
2. bid that will receive an amount of money from this function caller address and if it’s less the the previous bid it will return it to the sender, if it’s more it will keep it and return the previous stored amount to its sender. 3. showMax which will return the maximum bid so far, and the address of the highest bidder. Note: we can get rid of this function using the events that you’ll learn about in the next sprint.
4. buyAsset, this function will give an access to the highest bidder to the password of the asset (resemblance of buying the asset) and send the money to the beneficiary.
5. getAsset, this function will return the password to highest bidder.(Note: We need this function since the password can not be sent to the winner like ethers/tokens.

IV. (0-1) Knapsack Algorithem

Given weights and values of n items, put these items in a knapsack of capacity W to get the maximum total value in the knapsack. In other words, given two integer arrays val[0..n-1] and wt[0..n-1] which represent values and weights associated with n items respectively. Also given an integer W which represents knapsack capacity, find out the maximum value subset of val[] such that sum of the weights of this subset is smaller than or equal to W. You cannot break an item, either pick the complete item, or don’t pick it (0-1 property)
